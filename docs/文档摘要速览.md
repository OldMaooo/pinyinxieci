# 文档摘要速览

> 旨在让阅读者快速了解 `docs/` 目录下各文档的核心内容与使用场景。

| 文件 | 内容摘要 | 使用场景 | 备注 |
|---|---|---|---|
| 看拼音写词-需求汇总.md | 精炼的需求全貌：项目定位、核心流程、功能模块、数据模型、API 规范、UI/UX 要点与扩展规划。 | 作为产品/研发/大模型上手入口；快速了解当前实现范围。 | 最新整理版，建议从此文档开始阅读。 |
| 需求文档.md | 详尽的需求说明：覆盖手写输入、识别、题库管理、错题本、统计、数据导入导出等全部细节与阶段规划。 | 深入理解业务逻辑或补充细节；编写 PRD/技术方案时参考。 | 详版，内容较长，配合“汇总”使用。 |
| API使用说明.md | 描述百度手写识别 API 的配置方式、密钥管理、调用注意事项与调用限制。 | 配置或排查识别接口；部署前校验 API 信息。 | 需与现有 `api/` 代理实现保持一致。 |
| 识别判定标准.md | 对识别结果的判定规则、置信度阈值、判错策略等进行说明。 | 调整识别准确率或调试判定逻辑时参考。 | 可与 `API使用说明.md`、需求文档互相引用。 |
| 多应用平台架构规划.md | 针对未来多应用学习平台的整体规划：共享组件、样式、应用中心、部署策略等。 | 规划层参考；若扩展到多应用体系时抄底。 | 属于跨应用资料，非当下必读。 |
| 架构实施指南-通俗版.md | 以通俗语言解释多应用平台架构，面向非技术或新手的理解指南。 | 与团队沟通平台化愿景；帮助非技术同学理解。 | 与上文共同构成“规划参考”。 |
| 快速开始示例.md | 提供组件抽离、代码片段及使用示例，展示如何复用/扩展应用。 | 需要快速参考具体实现范例或编写教程时使用。 | 注意校对代码片段是否与当前版本一致。 |
| 项目结构.md | 说明仓库目录、主要文件功能和组织方式（含前端模块、脚本、数据等）。 | 了解仓库结构、定位代码位置。 | 建议合并更新“项目结构说明.md”内容。 |
| 项目结构说明.md | 与 `项目结构.md` 内容重合，包含更多解释说明。 | 历史遗留版本，用于补充描述。 | 建议与 `项目结构.md` 合并后归档。 |
| DEPLOY.md | 记录部署与发布流程（当前与其他文档有重叠）。 | 发布到 GitHub Pages/Vercel 或部署脚本时参考。 | 建议与“部署步骤”“快速启动”整合成统一部署指南。 |
| 部署步骤.md | 详细的部署步骤说明，涉及环境准备、构建与上线。 | 关注实际部署操作的人群。 | 与 `DEPLOY.md`、`快速启动.md` 合并后保留。 |
| 快速启动.md | 快速启动/本地运行指引。 | 开发环境启动、首次运行时参考。 | 建议整合进统一部署/开发指南。 |
| 启动代理服务器.md | 介绍如何在本地或线上启动识别 API 代理服务。 | 当需要本地调试或重新部署代理时使用。 | 可作为统一部署指南的附录。 |
| rules-draft.md | 早期规则草案，包含已废弃或不再适用的要求。 | 回溯历史决策或理解背景。 | 文首需标注“草案/历史”以免混淆。 |
| 文档清单与整合建议.md | 本仓文档审计结果：哪些保留、哪些合并、目标目录结构等。 | 执行文档整合、维护文档体系时参考。 | 与本文档互补：一个给决策，一个做摘要。 |
| 文档摘要速览.md | 当前文档（即本文），提供各文档用途及摘要。 | 新人或模型快速了解 docs 内容结构。 | 应与“文档清单与整合建议.md”共同维护。 |
| 语文认字-需求分析.md | 收录“语文认字”应用的需求分析，与本仓应用同属系列但非同仓实现。 | 需要跨应用协同时参考。 | 标注为“跨应用参考”，非本项目主文档。 |
