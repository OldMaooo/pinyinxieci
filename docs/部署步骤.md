# 部署步骤

## 方式一：通过命令行部署（推荐）

### 1. 在GitHub上创建仓库

1. 访问 https://github.com/new
2. 仓库名称：`chinese-handwriting-practice` （或你喜欢的名字）
3. 设置为 **Public**
4. 不要添加 README（我们已经有了）
5. 点击 "Create repository"

### 2. 添加远程仓库并推送

```bash
cd "/Users/mao/Documents/Coding/Development/Projects/Web/看拼音写词"

# 添加远程仓库（替换为你的GitHub用户名和仓库名）
git remote add origin https://github.com/YOUR_USERNAME/chinese-handwriting-practice.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

### 3. 启用GitHub Pages

1. 进入你的GitHub仓库页面
2. 点击 **Settings** → **Pages**
3. 在 **Source** 下选择 **GitHub Actions**
4. 保存设置

### 4. 等待自动部署

几分钟后，应用将在以下地址可用：
```
https://YOUR_USERNAME.github.io/chinese-handwriting-practice/
```

## 注意事项

### ⚠️ API代理问题

GitHub Pages 只能托管静态文件，**无法运行Node.js服务器**。

这意味着：
- 代理服务器（`proxy-server.js`）无法在GitHub Pages上运行
- 百度API的CORS问题仍然存在

**解决方案：**

1. **使用Vercel部署**（推荐）
   - Vercel支持Serverless Functions
   - 可以在 `api/` 目录下创建代理函数
   
2. **使用Netlify部署**
   - Netlify也支持Functions
   
3. **本地使用**
   - 在本地运行代理服务器
   - 直接在浏览器打开 `index.html`
